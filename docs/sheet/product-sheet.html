The problem → the solution

Why TeamUp

😖

Today: WhatsApp chaos

Every student needs a 4-person team under strict rules (grade mix, branch, gender). Finding teammates over group chats is messy, and you can't even tell if a team is valid until it's too late.

✅

TeamUp: rules that enforce themselves

An invalid team is impossible to form. Live filters, a searchable directory, faculty mentors, and an AI assistant — all on one page, on the college server.

Feature set

Everything a batch needs

🧩

Rule-enforced teams

Exactly 4 members; grade mix must be AABC/ABBC/ABCC/AACC/BBCC; CSE+AIML combine; ECE-only ECE teams. Checked at every step.

🔎

Browse & filter

Teams by branch, domain, open grade slot, "eligible to join". Live "6 of 9 teams" count.

🎓

Student directory

Search 1,500+ students by name/SRN; see who's free; filter by branch, gender, grade, interest.

👥

Requests & invites

Request to join, or invite eligible students. Any member can act; WhatsApp shared to vibe-check.

👑

Leadership tools

Approve, kick, disband, transfer the crown, team description — clear role permissions.

👨‍🏫

Faculty mentors

Real PES faculty with photos, emails, domain expertise. Searchable; attach to your team.

✨

AI Assistant

"Which team is Priya in?" · "Mentors for Blockchain?" — answers from live data, never hallucinated.

🔐

Privacy & security

CGPA never shown (only A/B/C); scrypt passwords; email-OTP signup & reset; activity log.

📱

Mobile-ready

Fully responsive — built for the phones most students will actually use.

Built to launch

Engineered for the whole batch

⚡ Load-tested

1,600 students / 350 teams — 20–40 ms/request; 200 concurrent requests served in ~3 s.

🛡️ Crash-safe

SQLite + WAL, ACID transactions. Verified with kill -9 mid-write — zero data loss.

💾 Backed up

Nightly SQLite snapshots, 14-day retention. One file to copy, restore, or hand off.

Why SQLite, not MongoDB

No database server to host

  • It's a file, not a service — nothing for IT to install, expose a port for, or maintain
  • Real relational SQL — tables, joins, ACID transactions
  • Portable further — exports to MySQL / PostgreSQL with zero rework
Tech stack
  • Node.js + Express — one service, no build step
  • SQLite (better-sqlite3) — one indexed file
  • Vanilla HTML / CSS / JS frontend
  • Ollama + Qwen for the assistant (CPU, no GPU)
  • Docker image · systemd · Caddy HTTPS
